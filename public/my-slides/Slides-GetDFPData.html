Accessing Financial Reports and Corporate Events Data using GetDFPData

Marcelo Perlin, Guilherme Kirch, Daniel Vancin

Unisinos/Porto Alegre - January 19, 2018

Introduction

  • Financial statements are at the heart of corporate finance studies
  • Commercial data vendors offers paid solutions, but with some restrictions for large scale access
  • B3 provides public access to its datasets in three different systems: DFP, ITR, FRE and FCA, but with several limitation for large scale access
  • GetDFPData provides an open R interface to all financial statements distributed by B3 and CVM

Motivation

  • A look into past publications in Brazil:

Features of GetDFPData

Made by researchers, to researchers

  • Provides a reference table with up to date information about companies:
    • available reports, sectors, governance listing segments, stock tickers
  • Cleans and adjusts the data
    • Different accounting formats are merged using matching tools
    • Errors in the data are fixed, when possible, or discarted

  • Always tabular format (tidy data principles (Wickham 2014)) and always the correct class in R

  • R and web/shiny interface

  • Open and free!

A closer look at gdfpd.GetDFPData

Function gdfpd.GetDFPData provides many options to the user. Here we will describe and explain its features.

  • name.companies: Official names of companies to get financial reports (e.g. ELETROPAULO METROPOLITANA EL.S.PAULO S.A).
  • first.date: First date (YYYY-MM-DD) to download data. Character or Date. E.g. first.date = 2010-01-01.
  • last.date: Last date (YYYY-MM-DD) to download data. Character or Date. E.g. last.date = 2017-01-01.
  • inflation.index: Sets the inflation index to use for finding inflation adjusted values of all reports. Possible values are dollar (default) or IPCA, the brazilian main inflation index. When using IPCA, the base date is set as the last date found in the DFP dataset.
  • max.levels: Sets the maximum number of levels of accounting items in financial reports such as assets and liabilities (default = 3). If set to Inf, will return all available accounting itens.
  • folder.out: Folder where to download and manipulate the zip files. Default = tempdir()
  • do.cache: Logical for using a cache system or not. Default = TRUE
  • cache.folder: Folder where to cache all processed information. Default = file.path(getwd(),DFP Cache Folder)
  • max.dl.tries: Maximum number of attempts for dowloading files

Exporting financial data

The package also includes function gdfpd.export.DFP.data for exporting the financial data in the Excel or csv format.

Next we present an example of saving the output as an Excel file.

my.basename <- 'MyExcelData'
my.type.export <- 'xlsx' # only supported so far

gdfpd.export.DFP.data(df.reports = df.reports, 
                      base.file.name = my.basename,
                      type.export = my.type.export)
